## Findings

### 🔴 **CRITICAL**: No HTTP → HTTPS redirect on any site

- **Category:** security
- **Affected:** `books.` `spf.` `argocd.` `headlamp.` `traefik.` .minhnguyenle.net

All five domains return **404** when accessed over HTTP. Users who type `http://` or follow old links get a dead page. Worse, some clients (OPDS readers, older browsers) default to HTTP and silently fail.

**Recommendation:** Add a Traefik middleware or IngressRoute that redirects HTTP→HTTPS:
```yaml
apiVersion: traefik.io/v1alpha1
kind: Middleware
metadata:
  name: redirect-https
  namespace: kube-system
spec:
  redirectScheme:
    scheme: https
    permanent: true
```

### 🔴 **CRITICAL**: spf-mmo login has no rate limiting

- **Category:** security
- **File:** `spf-mmo server` — `/api/auth/login` endpoint

The login endpoint creates new user accounts with **no rate limiting**. Five requests in rapid succession all returned 200 with new accounts. An attacker can flood the endpoint to:
- Spam-create thousands of bot accounts
- Exhaust database storage
- Degrade service for legitimate users

**Recommendation:** Add rate limiting to the login route:
```typescript
import rateLimit from 'express-rate-limit';
const loginLimiter = rateLimit({
  windowMs: 15 * 60 * 1000, // 15 minutes
  max: 10, // 10 attempts per window
  message: { error: 'Too many login attempts' }
});
app.use('/api/auth/login', loginLimiter);
```

### 🟡 **WARNING**: Headlamp service account has cluster-admin

- **Category:** security
- **File:** `k3s_lab/headlamp/resources/cluster-admin-users.clusterrolebinding.yaml`

The `headlamp-admin` ServiceAccount is bound to `cluster-admin` ClusterRole — unrestricted access to the entire k3s cluster. If Headlamp authentication is breached, the attacker gets full cluster control.

**Recommendation:** Restrict to read-only or a limited set of permissions unless full admin is genuinely needed via the web UI.
